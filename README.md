## 🌍 Общее описание

**Twitch Pixel Arena** — это кооперативный автобатлер с пиксельной 2D-графикой, который работает прямо в браузере. Стример открывает страницу, подключается к своему Twitch-каналу, и зрители через чат-команды присоединяются к игре. Персонажи зрителей сражаются автоматически (без прямого управления) с нарастающими волнами монстров.

### Ключевые особенности:

- **Автобатлер**: Персонажи игроков действуют самостоятельно (ИИ выбирает цель, атакует, уклоняется).
- **Twitch-интеграция**: Подключение к реальному чату через WebSocket IRC.
- **Система волн**: Бесконечная эскалация сложности с боссами каждые 10 волн.
- **Голосование за реликвии**: Между волнами зрители голосуют за баффы/дебаффы, влияя на ход игры.
- **4 класса персонажей**: Воин, Лучник, Маг, Целитель — с уникальными механиками.
- **Пиксельная графика**: Все спрайты генерируются процедурно в коде (без внешних ассетов).

## 🏗 Архитектура и порядок загрузки

Скрипты загружаются последовательно в `index.html` и **зависят от порядка**:

```
config.js → sprites.js → particles.js → twitch.js → entities.js → game.js → ui.js
```

1. **`config.js`** — глобальный объект `CONFIG` (характеристики, баланс, реликвии).
2. **`sprites.js`** — объект `Sprites`, генерирует все пиксельные спрайты в offscreen canvas при `init()`.
3. **`particles.js`** — класс `ParticleEngine`, обрабатывает визуальные эффекты (кровь, искры, текст).
4. **`twitch.js`** — класс `TwitchConnection`, WebSocket IRC клиент + симулятор.
5. **`entities.js`** — иерархия классов: `Entity` → `Player`, `Enemy` → `Boss`, `Projectile`, `HealthPotion`, `FirePuddle`.
6. **`game.js`** — класс `Game`, ядро: управляет состоянием, волнами, рендерингом, командами.
7. **`ui.js`** — точка входа: `DOMContentLoaded`, создает экземпляр `Game`, запускает `requestAnimationFrame` цикл, подключает DOM-элементы.

### Паттерн экспорта

Каждый модуль проверяет `typeof module !== 'undefined'` и экспортирует через `module.exports` (для потенциальных тестов в Node.js) или вешает на `window` (для браузера).

## 🖥 Игровые экраны и пользовательский поток

Приложение состоит из 3 экранов, переключаемых функцией `showScreen()`:

### 1. Лобби (`#lobby-screen`)

- Стример вводит имя Twitch-канала и подключается. **Введенное имя канала сохраняется в LocalStorage и автоматически восстанавливается при перезагрузке страницы.**
- Зрители отправляют `!join [класс]` в чат и появляются в списке ожидания.
- На экране отображается количество игроков каждого класса в реальном времени для быстрой оценки баланса классов.
- Стример нажимает **«Начать битву!»** для старта.
- Если никто не присоединился, автоматически создаются 4 бота (по одному на каждый класс).
- Кнопка **«Симулировать вход бота»** добавляет тестового зрителя.

### 2. Игровой экран (`#game-container`)

- Полноэкранный Canvas (1200×800 виртуальных пикселей, масштабируемый под окно).
- HUD сверху: номер волны, количество оставшихся до конца волны врагов (или таймер голосования), количество живых, счёт.
- Панель симулятора чата справа внизу (сворачиваемая).
- Логи Сражения слева внизу: показывает в реальном времени никнеймы погибших и воскрешенных игроков (очищается каждую волну).
- Чередование фаз: **Бой** → **Голосование** → **Бой** → ...
- **Мгновенное присоединение**: Новые игроки могут присоединяться к игре командой `!join` в любой момент. Если бой уже идет, они мгновенно спавнятся в случайном месте на арене с визуальным эффектом телепортации. Если идет фаза голосования, они заспавнятся при старте следующей волны.
- **Случайное распределение при старте волны**: В начале каждой новой волны все игроки (как выжившие, так и возрождённые) распределяются по арене случайным образом для предотвращения стартового скучивания.

### 3. Экран Game Over (`#gameover-screen`)

- Показывается, когда **все игроки погибли**.
- Отображает: пройденные волны, время выживания, итоговый счёт, суммарное лечение и количество воскрешений, совершенных целителями.
- Окно вытянуто на всю высоту экрана для удобного отображения максимального количества никнеймов игроков.
- Таблица статистики каждого зрителя: класс, уровень, убийства, нанесённый урон, вылеченное HP.
- Кнопка **«Вернуться в лобби»** для перезапуска.

### Диаграмма состояний

```
lobby → playing ⇄ voting → playing → ... → gameover → lobby
```

- `lobby` — ожидание игроков
- `playing` — активный бой (волна)
- `voting` — голосование за реликвию (30 секунд)
- `gameover` — все погибли

## ⚔️ Классы игроков

Каждый зритель выбирает один из 4 классов. Класс определяет характеристики, стиль боя и визуал спрайта.

| Характеристика | Воин (warrior) | Лучник (archer) | Маг (mage) | Целитель (healer) |
|---|---|---|---|---|
| **HP** | 120 | 60 | 50 | 70 |
| **Скорость** | 1.8 | 2.3 | 1.6 | 2.0 |
| **Урон** | 12 | 8.4* | 20 | 7 (лечение) |
| **Дальность** | 45 (ближний бой) | 250 | 220 | 100 |
| **Кулдаун (кадры)** | 50 (~0.8 сек) | 40 (~0.65 сек) | 90 (~1.5 сек) | 60 (~1.0 сек) |
| **Защита** | 5 | 1 | 0 | 2 |
| **Цвет** | Красный `#e74c3c` | Зелёный `#2ecc71` | Фиолетовый `#9b59b6` | Жёлтый `#f1c40f` |
| **Тип атаки** | Взмах мечом (AoE конус) | Стрела (снаряд, самонаведение) | Огненный шар → лужа (AoE) | Сфера лечения (снаряд) |
| **Рост за уровень** | +15 HP, +1.5 DMG, +1 DEF | +6 HP, +1.0 DMG, +0.05 SPD | +5 HP, +2.5 DMG, +5 RNG | +8 HP, +1.5 HEAL, +0.02 SPD |

*\* Лучники обладают врожденным шансом критического удара 10% (критический удар наносит +100% урона, т.е. x2).*

### Поведение ИИ по классам

- **Воин**: Бежит к ближайшему врагу, бьёт мечом в AoE-конусе перед собой (радиус 60 пикселей). **Имеет 20% шанс полностью заблокировать входящий урон (сопровождается надписью «БЛОК!»). Если уровень здоровья воина падает ниже 5%, он прекращает атаковать и пытается убежать от врагов.**
- **Лучник**: Стреляет на расстоянии. Если враг подходит вплотную (<35 px), убегает. После бегства — пауза в 1 секунду перед следующим выстрелом.
- **Маг**: Аналогичен лучнику, но выпускает файрболы, которые создают **огненные лужи** при попадании.
- **Целитель**: Не бежит к противникам, а всегда следует за ближайшим живым союзным игроком на безопасном расстоянии. Ищет союзника или самого себя с наибольшей потерей HP (в процентах). Если кто-то ранен, выпускает в него самонаводящуюся лечащую сферу (при самоисцелении сила лечения снижается на 50%). Если все здоровы, просто держится рядом с союзниками. **При опасном приближении врага (<45 пикселей) целитель прекращает действия и пытается убежать от него.** Получает опыт (XP) в размере 50% от величины отхила при лечении союзников (за самолечение опыт не дается). Способен воскрешать павших союзников (когда те превращаются в могилы) раз в 20 секунд: для этого целитель должен подойти к могиле на расстояние действия заклинания; воскрешенный союзник возвращается к жизни с 50% HP.

### Смена класса

Зритель может сменить класс командой `!class [класс]`. Смена вступает в силу **в начале следующей волны**. Ограничение: **1 смена за волну**.

## 👾 Система врагов

### Типы монстров

| Тип | HP | Скорость | Урон | Кулдаун | Масштаб | XP | Очки | Когда появляется |
|---|---|---|---|---|---|---|---|---|
| **Слизень** (slime) | 14 | 1.0 | 7 | 60 | 0.8x | 10 | 10 | С 1-й волны |
| **Гоблин** (goblin) | 21 | 2.0 | 10 | 45 | 0.9x | 15 | 20 | С 3-й волны |
| **Гоблин-метатель** (goblin_stone) | 17 | 1.5 | 8 | 80 | 0.9x | 20 | 30 | С 3-й волны (дальний бой) |
| **Бычара** (bull) | 80 | 1.0 | 0 | 180 | 1.4x | 30 | 50 | С 3-й волны |
| **Скелет** (skeleton) | 31 | 1.2 | 15 | 70 | 1.0x | 25 | 40 | С 6-й волны |
| **Орк** (orc) | 77 | 0.8 | 24 | 100 | 1.3x | 60 | 100 | С 10-й волны |
| **Рейд-Босс** (boss) | 1050* | 1.1 | 42 | 120 | 2.2x | 500 | 1000 | Каждую 10-ю волну |

### Масштабирование по волнам

Все характеристики монстров масштабируются с каждой волной:

```
Множитель = 1.0 + (номерВолны - 1) × 0.15
```

Пример: на 10-й волне монстры имеют `1.0 + 9 × 0.15 = 2.35x` от базовых HP и урона.

### Масштабирование сложности и количества от лобби

Сложность и количество врагов масштабируются пропорционально количеству игроков в лобби:

1. **Количество врагов**:
   ```
   кол-во = (baseCount + (волна - 1) × countPerWave) × количествоИгроковВЛобби
   ```
   * Базовое (половинное): 4 монстра + 2 за каждую волну.
   * Количество врагов увеличивается прямо пропорционально количеству участников.

2. **Характеристики врагов (сложность)**:
   * **Здоровье (HP)** монстров масштабируется пропорционально количеству игроков в лобби:
     ```
     МножительHP = МножительВолны × количествоИгроковВЛобби
     ```
   * **Урон** монстров НЕ зависит от количества игроков в лобби и увеличивается только с номером волны (`МножительВолны`), предотвращая мгновенные убийства игроков в больших лобби.

### Рейд-Босс

- Появляется каждые 10 волн (10, 20, 30, ...).
- HP масштабируется от количества живых игроков: `+45%` за каждого игрока.
- **Уникальная способность — «Землетрясение»**: каждые 6 секунд наносит AoE-урон (60% от базового) в радиусе 160 px + тряска камеры.
- Наследует класс `Boss extends Enemy`.

### Бычара

- Появляется начиная с 3-й волны.
- **Уникальная способность — «Разбег»**: раз в 3 секунды (180 кадров) выбирает ближайшего живого игрока и устремляется к нему с увеличенной в 4.5 раза скоростью.
- При столкновении с игроком прекращает разбег, сотрясает экран (эффект «БАМС!») и раскидывает всех игроков в радиусе 160 пикселей сильным физическим импульсом отбрасывания (knockback).
- Не наносит урона (базовый урон равен 0). Служит для дезориентации и разгона игроков на арене.

### Спавн врагов

- Враги появляются за пределами экрана (сверху, снизу, слева, справа — случайно).
- Между спавнами — задержка в 90 кадров (~1.5 секунды).
- На волнах с боссом первым спавнится босс из центра сверху.

## ⚔️ Боевая система

### Расчёт урона

```javascript
актуальныйУрон = max(1, round(базовыйУрон - защитаЦели))
```

- Урон рандомизируется: `базовый × (0.8 + random × 0.4)` — разброс ±20%.
- Защита вычитается после рандомизации.
- Минимальный урон всегда 1.

### Механика разталкивания (Separation)

Каждый кадр все активные сущности проверяются попарно. Если расстояние между двумя сущностями меньше минимального (`(w1 + w2) / 2 × 0.75`), они отталкиваются друг от друга с учётом веса:

- Босс имеет вес 4 (его почти невозможно сдвинуть).
- Воин имеет вес 1.5.
- Остальные — вес 1.0.

### Тряска камеры

Вызывается при ударе босса (Землетрясение) и через метод `game.shake(amount)`. Значение тряски экспоненциально затухает (`×0.9` каждый кадр). Реализуется через `ctx.translate()` со случайным смещением.

## 🏹 Система снарядов

Класс `Projectile` обрабатывает все летящие объекты.

### Типы снарядов

| Тип | Владелец | Цель | Скорость | Визуал | При попадании |
|---|---|---|---|---|---|
| `arrow` | Лучник | Враг | 7.5 | Стрела (16×8 px) | Прямой урон + вампиризм |
| `fireball` | Маг | Враг | 4.2 | Красная сфера | Урон + создание огненной лужи (FirePuddle) |
| `healball` | Целитель | Союзник | 5.0 | Зелёная сфера | Лечение цели |

### Поведение

- **Самонаведение**: Снаряд корректирует направление полёта к цели каждый кадр.
- **Если цель умерла**: Снаряд продолжает лететь по последнему вектору.
- **Магические следы**: Каждый кадр с шансом 30% спавнятся частицы-искры за снарядом.
- Удаляется при вылете за пределы арены (+100 px от каждого края).

### Огненная лужа (FirePuddle)

- Создаётся при попадании файрбола мага.
- **Базовый радиус**: 50 px (масштабируется реликвиями).
- **Длительность**: 120 кадров (~2 сек, масштабируется реликвиями).
- **Периодический урон**: каждые 15 кадров (4 раза/сек), 35% от урона мага.
- Визуально — полупрозрачный оранжевый эллипс с искрами огня.
- Плавно растворяется в последние 30 кадров жизни.

## 🗳 Система реликвий и голосования

### Фаза голосования

Активируется автоматически после зачистки каждой волны:

1. `gameState` переключается на `'voting'`.
2. Выбираются 4 случайные реликвии с учётом весов редкости.
3. Таймер: **30 секунд** (реальное время).
4. Зрители голосуют, отправляя `1`, `2`, `3` или `4` в чат.
5. Каждый зритель может проголосовать **один раз** (`votedUsers: Set`).
6. По истечении таймера побеждает вариант с наибольшим числом голосов. Если никто не голосовал — выбирается случайно.

### Редкости реликвий

| Редкость | Цвет | Вес (шанс) |
|---|---|---|
| Обычная (common) | Серый `#95a5a6` | 60 |
| Редкая (rare) | Синий `#3498db` | 25 |
| Эпическая (epic) | Фиолетовый `#9b59b6` | 12 |
| Легендарная (legendary) | Золотой `#f1c40f` | 3 |

### Полный каталог реликвий

#### Классовые баффы

| ID | Название | Редкость | Эффект |
|---|---|---|---|

### Система модификаторов

Применённые реликвии накапливаются в объекте `relicModifiers`:

```javascript
{
  warrior: { maxHp: 1.25, damage: 1.2 },  // Мультипликативные модификаторы
  archer: {},
  mage: {},
  healer: {},
  all: { speed: 1.15 },
  enemies: { speed: 0.8 },  // Дебаффы врагов
  mechanics: {
    lifesteal: 0.10,   // Вампиризм
    thorns: 0.30,       // Шипы
    berserk: true        // Берсерк
  },
  xpMul: 1.5              // Множитель опыта
}
```

- При получении реликвии все живые игроки мгновенно пересчитывают характеристики через `applyRelicModifiers()`.
- Мультипликаторы **умножаются** (два баффа +25% HP = ×1.5625).
- Аддитивные модификаторы **складываются** (defense +4, range +50).

## 📈 Система уровней и опыта

### Формула XP для уровня

```
xpNeeded = XP_BASE + level × XP_FACTOR
         = 50 + level × 35
```

| Уровень | XP для следующего |
|---|---|
| 1 → 2 | 50 |
| 2 → 3 | 85 (50 + 2×35) |
| 3 → 4 | 120 |
| 5 → 6 | 225 |
| 10 → 11 | 400 |

### При повышении уровня

1. Полное лечение (HP = maxHP).
2. Пересчёт характеристик с учётом бонусов роста класса + реликвий.
3. Визуальный эффект: золотой текст «УРОВЕНЬ N!» + искры.

## 🎨 Визуальный пайплайн

### Рендеринг (каждый кадр)

```
1. Применить тряску камеры (ctx.translate)
2. Нарисовать фон (тёмно-зелёная трава #3b7a57 + текстура травинок)
3. Нарисовать запечённую кровь на земле (bloodCanvas)
4. Нарисовать огненные лужи
5. Y-сортировка ВСЕХ объектов → отрисовка снизу вверх:
   - Декорации (деревья, кусты, камни)
   - Игроки (живые — спрайты; мёртвые — могильные плиты с серым ником)
   - Враги
   - Лут (зелья)
6. Снаряды (стрелы, файрболы, сферы лечения) — поверх всех объектов
7. Динамические частицы + плавающий текст
8. UI голосования (если gameState === 'voting')
```

### Y-сортировка

Все объекты помещаются в массив `sortedObjects` с ключом `y` (нижняя граница объекта). Сортировка по `y` обеспечивает 2.5D-эффект глубины: объекты ниже на экране рисуются поверх тех, что выше.

### Процедурные спрайты (Sprites)

Все спрайты генерируются при инициализации через offscreen `<canvas>`:

- **Игроки**: 5 кадров на класс (idle, walk0, walk1, attack0, attack1) × 4 класса = 20 спрайтов.
- **Враги**: 5 кадров на тип × 5 типов = 25 спрайтов. Слизень имеет уникальную форму (без гуманоидного тела).
- **Снаряды**: 3 спрайта (стрела, огненная сфера, лечебная сфера).
- **Предметы**: зелье, могильная плита.
- **Декорации**: дерево (64×64), куст (32×32), камень (32×32).

Все холсты имеют `imageSmoothingEnabled = false` для чёткого пиксель-арта.

### Декорации арены

При инициализации (`game.init()`) на арене случайно размещаются:
- 8–12 деревьев
- 15–22 куста
- 10–14 камней

Декорации не являются препятствиями (нет коллизий), служат чисто визуальному оформлению.

### Анимация персонажей

- Переключение кадров каждые 10 тиков (animTimer).
- Позы: `idle`, `walk`, `attack`.
- Ходьба: вертикальный bob-эффект (смещение -2 px вверх на кадре walk1).
- Все спрайты зеркалятся горизонтально через `ctx.scale(-1, 1)` при движении влево.

### Визуальные ауры

Игроки с усиленными характеристиками получают косметические ауры:
- **Синий щит** — повышенная защита (вращающиеся синие пиксели вокруг).
- **Красное свечение** — повышенный урон.
- **Зелёный ветер** — повышенная скорость.
- **Золотая аура** — легендарная реликвия (вампиризм / шипы / берсерк).

## ✨ Система частиц

Класс `ParticleEngine` обрабатывает все динамические визуальные эффекты.

### Типы частиц

| Тип | Описание | Гравитация | Трение | Время жизни |
|---|---|---|---|---|
| `blood` | Брызги крови при уроне | 0.15 | 0.96 | 30–60 кадров |
| `spark` | Вспышки магии / следы снарядов | 0 | 0.95 | 15–30 кадров |
| `aura` | Частицы ауры вокруг игрока | 0 | 1.0 | 20–40 кадров |

### Запечённая кровь

- Частицы `blood` при "падении на землю" (после 15 кадров полёта, шанс 15%) рисуют **пятно** на отдельном offscreen `bloodCanvas`.
- Этот холст рисуется каждый кадр поверх фона (слой крови на полу).
- Каждые 300 кадров (~5 секунд) пятна крови немного тускнеют (`destination-out` с альфой 0.05) — чтобы арена не забивалась кровью.

### Плавающий текст (FloatingText)

- Цифры урона — красные, мелкий шрифт, жизнь 45 кадров.
- Критический урон — оранжевый, крупнее, жизнь 65 кадров.
- Уровень — золотой, самый крупный (`УРОВЕНЬ N!`), жизнь 120 кадров.
- Все тексты медленно поднимаются вверх, затухая.

## 📺 Интеграция с Twitch

### Подключение

Класс `TwitchConnection` подключается к IRC через WebSocket:

```
wss://irc-ws.chat.twitch.tv:443
```

- **Анонимная авторизация**: `NICK justinfanXXXXX` (случайный номер). Без OAuth-токена — только чтение.
- **Запрос тегов**: `CAP REQ :twitch.tv/tags twitch.tv/commands` — для получения цвета ника и display-name.
- **PING/PONG**: Автоматический ответ на `PING` для поддержания сессии.

### Парсинг сообщений

Regex для PRIVMSG с тегами:

```regex
^(?:@([^ ]+) )?:([^!]+)![^ ]+ PRIVMSG #[^ ]+ :(.+)$
```

Извлекаются:
- `display-name` — отображаемое имя.
- `color` — цвет ника в Twitch. Если не задан — генерируется по хэшу имени.

### Коллбэки

- `onCommandCallback(username, command, args, userColor)` — игровая команда (обрабатывается в `game.handleCommand`).
- `onLogCallback(username, message, userColor, isSimulated)` — лог для UI чата.
- `onStatusChangeCallback(statusText, isError)` — статус подключения.

## 💬 Чат-команды

### Основные команды

| Команда | Алиасы | Описание |
|---|---|---|
| `!join [класс]` | `!войти`, `!играть` | Присоединиться к игре. Без класса — случайный |
| `!class [класс]` | `!класс` | Сменить класс (вступает в силу в следующей волне) |
| `1` / `2` / `3` | `!1` / `!2` / `!3` | Голосовать за реликвию |

### Доступные названия классов

| Ключ | Русский | Английский |
|---|---|---|
| warrior | воин | warrior |
| archer | лучник | archer |
| mage | маг | mage |
| healer | целитель, хилер, клирик | healer, cleric |

### Логика обработки команд

Все команды проходят через `game.handleCommand()`:

- **`!join`** в лобби → добавляет в `lobbyUsers` (Map: username → classType).
- **`!join`** во время игры → добавляет в `pendingSpawnUsers` (спавнится в начале следующей волны).
- **`!class`** в лобби → обновляет класс в `lobbyUsers`.
- **`!class`** во время игры → устанавливает `player.pendingClassChange` (1 раз за волну).
- **Голоса** → принимаются только в `gameState === 'voting'`, по 1 голосу на зрителя.

## ⚠️ Известные особенности

1. **Дроп зелий**: Параметр `HEAL_POTION_DROP_CHANCE` задан в конфиге, класс `HealthPotion` полностью реализован, но фактический вызов создания зелья при убийстве врага ещё не подключён в `game.js`. Механика готова к включению.
2. **OBS-режим**: CSS скрывает панели, но фон Canvas `#3b7a57` остаётся непрозрачным — для полной прозрачности арены нужна доработка в коде рендеринга.
3. **Нет Node/npm**: Проект не использует бандлер, все скрипты подключены через `<script>` теги с жёсткой последовательностью.
4. **Нет сохранений**: Прогресс между сессиями не сохраняется.
